# 组件的封装

**封装组件的时候，必须在组件的灵活性和易用性之间/稳定性之间做出一个选择**

组件越灵活，相应的越不稳定，也越不易用；

越是简单的组件，灵活性和定制性就不够。

参考Lin ui  可以把小程序里的Page都换成Component，因为Component更加灵活

**组件的意义是什么？**

组件从三个方面：样式、骨架、业务逻辑/行为上对代码进行封装，方便开发者在日后重用/复用样式、骨架和业务逻辑

**为了保证组件的灵活性，可以采取以下措施进行改造：**

以小程序的组件为例：

1、提供外部样式类(样式)

2、slot插槽的使用（骨架）

封装组件的时候，要设置好默认值
